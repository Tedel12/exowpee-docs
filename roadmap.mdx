This roadmap outlines our vision, current status, near-term priorities, and longer-term ambitions for the Exowpee API.  
We are committed to building a modern, scalable, and developer-friendly platform that powers businesses across Africa.

Last updated: February 2026

## Current Status (v1.0 – Q1 2026)

- **Fully documented core modules**  
  - IAM (Authentication, Roles, Permissions, Profiles)  
  - Catalog (Products, Variants, Categories, Vendors, Locations)  
  - Customer (CRUD, profiles, history)  
  - Order (Lifecycle, actions: confirm/reject/complete/cancel, cashing)  
  - Ledger (Basic structure, automatic entries from sales & purchases)  
  - Team / Companies (Multi-company, branches, logos)

- **OpenAPI specification** imported and live in API Reference  
- **Beginner-friendly documentation** with cURL examples, step-by-step tutorials, permissions explanations  
- **Multi-tenant architecture** enforced everywhere (company_id scoping)  
- **Deployed documentation site** (Mintlify + Vercel)

## Near-term Priorities (Q2–Q3 2026)

### Q2 2026 – Completeness & Developer Experience

- Finish & document remaining modules  
  - **Settings** (pricing plans, payment methods, units, roles CRUD)  
  - **Reports** (low stock, sales by product, best sellers, credit sales, product activity)  
  - **Stock & Transfer** (stock list, regularizations, transfers between locations)  
  - **Purchase** (full purchase lifecycle, partial deliveries)

- Add **Postman Collection** official download  
  - Importable collection with examples for all endpoints  
  - Pre-configured environments (Production / Local)

- Implement & document **token refresh** flow  
  - Refresh token endpoint  
  - Best practices for long-lived sessions

- Add **Error Reference** centralized page  
  - All common 4xx/5xx codes  
  - Causes, messages, how to fix  
  - Examples of validation errors (422)

- Introduce **Webhooks** (early version)  
  - Events: order.created, order.confirmed, payment.received, stock.low  
  - Documentation + signature verification

### Q3 2026 – Polish & Ecosystem

- **SDKs** (JavaScript/TypeScript + Python)  
  - Official client libraries with authentication, retry logic, type safety

- **Rate Limiting & Quota** documentation  
  - Limits per endpoint / per company  
  - Headers (X-RateLimit-Remaining, etc.)

- **Testing & Sandbox** environment  
  - Public sandbox server with fake data  
  - Guide for integration testing

- **Versioning strategy**  
  - v1 → v2 transition plan  
  - Deprecation policy

## Long-term Vision (2027+)

- **GraphQL alternative** (in addition to REST)  
  - More flexible queries for complex use cases

- **Embedded iFrame / White-label** capabilities  
  - Allow partners to embed Exowpee checkout/widgets

- **Advanced reporting & analytics**  
  - Dashboard-style aggregated data  
  - Export CSV/PDF

- **Mobile SDKs** (Flutter, React Native)  
  - For POS apps, delivery tracking

- **AI-assisted features** (future)  
  - Product description generation  
  - Sales forecasting  
  - Anomaly detection in orders/stock
